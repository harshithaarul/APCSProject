Chapter 1: Intro to Java


Basic computing concepts

  • Algorithm: step-by-step description of how to accomplish a task
  • Program: list of instructions to be carried out by a computer
  • Digital: based on numbers that increase in discrete increments, such as integers
  • Binary number: number composed of just 0s and 1s, AKA base-2 numbers
  • Program execution: act of carrying out instructions contained in a program
  • Compiler: program that translates a program you wrote into another language sp that the computer can read the program
  • Java Virtual Machine (JVM): a theoretical computer whose machine language is the set of Java bytecodes
  • Java runtime: program that executes compiled Java bytecodes
  • How to convert between bases:
    • Tutorial Video

Naming Java files

  • Always name files starting with capital letters
  • Class names are always the same as file names

Good programming style

  • Put class and method headers on lines by themselves
  • Put no more than one statement on each line
  • Indent properly
    • When opening brace appears, increase indentation of the lines that follow it
    • When closing brace appears, reduce indentation
    • Indent using 4 spaces
  • Use blank lines to separate parts of the program(e.g. methods)

Most common escape sequences

  • tab character: \t
  • new line character: \n
  • quotation mark: \"
  • backslash character: \\

Common programming/syntax errors

  • Statements must ALWAYS end with ";"
  • Quotations, parentheses, brackets should ALWAYS be in pairs
  • Avoid using Java keywords when naming variables found on pg20

Static methods

  • Static method: a block of Java statements that is given a name
    • Ex.: public static void methodName(){}
      • public indcates this method is available to be used by all parts of your program
      • static indcates that this method can be called without first creating an instance object (more details in Chapter 8)
      • void indicates this method executes statements but does not produce any value
  • Method call: a command to execute another method, which causes all statements inside that method to execute
    • Methods can also call other methods
      • But having a static method calling itself can create a runtime error
